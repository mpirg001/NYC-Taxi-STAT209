NYC Taxi Trip Total Payment Modellling

Objective:

  • How are location, trip characteristics, payment type, trip duration, and fare-related charges associated with the total payment amount for NYC taxi trips?

Data

  • Taxi: January 2024 NYC Yellow Taxi Trip Dataset.

\[2,964,624 \times 19 \longrightarrow \text{ random subsample: } 296,462 \times 19\]

Category Variables
Trip and provider information VendorID, store_and_fwd_flag
Time information tpep_pickup_datetime, tpep_dropoff_datetime
Location information PULocationID, DOLocationID
Trip characteristics passenger_count, trip_distance, RatecodeID
Payment information payment_type
Fare and related charges fare_amount, extra, mta_tax, tip_amount, tolls_amount, improvement_surcharge, total_amount, congestion_surcharge, Airport_fee
  • zones: \(\quad 265 \times 4\)

Research Questions

How are location, trip characteristics, payment type, trip duration, and non-fare charges associated with the total payment amount for NYC taxi trips?

Specific research questions are:

  1. Which variables are most strongly related to the total payment amount for a taxi trip?

  2. How do trip distance and trip duration affect the total payment amount?

  3. Do pickup and drop-off locations help explain differences in total payment amount?

  4. Are airport trips, rush-hour trips, or different payment types associated with higher total payment amounts?

EDA

Modeling Objective

  • The goal is to predict log_total_amount, defined as

\[ \log(1 + \text{total\_amount}) \]

using trip characteristics, payment type, location variables, and time indicators.

  • A clean modeling data set and one common train-test split are used for all models.

  • OLS is fitted first as the baseline model and evaluated using RMSE, test-set \(R^2\), residual diagnostics, the Breusch-Pagan test, and VIF/GVIF.

  • Because OLS shows nonconstant variance, robust standard errors and a high-leverage sensitivity check are considered.

  • Ridge, LASSO, PCR, WLS regression, and \(C_p\)-selected OLS are compared using the same test-set RMSE and test-set \(R^2\).

  • The final model is selected based on prediction accuracy, simplicity, and interpretability.

\[ \begin{aligned} \log(1+\text{total\_amount}) = \beta_0 &+ \beta_1 \log(1+\text{trip\_distance}) \\[0.4em] &\quad + \beta_2 \log(1+\text{trip\_duration\_min}) \\[0.4em] &\quad\quad + \beta_3 \text{passenger\_count} \\[0.4em] &\quad\quad\quad + \beta_4 \text{tip\_amount} \\[0.4em] &\quad\quad\quad\quad + \beta_5 \text{tolls\_amount} \\[0.4em] &\quad\quad\quad\quad\quad + \beta_6 \text{weekend} \\[0.4em] &\quad\quad\quad\quad\quad\quad + \beta_7 \text{rush\_hour} \\[0.4em] &\quad\quad\quad\quad\quad\quad\quad + \beta_8 \text{nighttime} \\[0.4em] &\quad\quad\quad\quad\quad\quad\quad\quad + \boldsymbol{\gamma}^\top \text{payment\_type} \\[0.4em] &\quad\quad\quad\quad\quad\quad\quad\quad\quad + \boldsymbol{\delta}^\top \text{pickup\_borough} \\[0.4em] &\quad\quad\quad\quad\quad\quad\quad\quad\quad\quad + \boldsymbol{\eta}^\top \text{dropoff\_borough} + \varepsilon . \end{aligned} \]

Ridge Regression

\[\hat{\boldsymbol{\beta}}_{\text{ridge}} = \arg\min_{\boldsymbol{\beta}} \left\{ \sum_{i=1}^{n} \left(y_i - \mathbf{x}_i^\top \boldsymbol{\beta}\right)^2 + \lambda \sum_{j=1}^{p} \beta_j^2 \right\}, \qquad \lambda \ge 0.\]

Ridge regression is evaluated mainly through prediction performance. Since the adjusted GVIF values did not show severe multicollinearity, Ridge is not expected to greatly improve over OLS.

LASSO Regression

LASSO is another regularized model. Unlike Ridge, LASSO can shrink some coefficients exactly to zero, so it can perform variable selection.

\[ \hat{\boldsymbol{\beta}}_{\text{lasso}} = \arg\min_{\boldsymbol{\beta}} \left\{ \sum_{i=1}^{n} \left(y_i - \mathbf{x}_i^\top \boldsymbol{\beta}\right)^2 + \lambda \sum_{j=1}^{p} |\beta_j| \right\}, \qquad \lambda \ge 0. \]

LASSO is useful if it achieves similar prediction accuracy to OLS while keeping a simpler model with fewer active predictors.

Principal Component Regression

Principal Component Regression (PCR)

PCR replaces correlated predictors with uncorrelated principal components.

Each principal component is a weighted combination of the original predictors:

\[ PC_k = v_{k1}X_1 + v_{k2}X_2 + \cdots + v_{kp}X_p \]

\[ \text{Correlated: $X=[X_1, X_2, \dots, X_p]$ $\xrightarrow{\text{PC Analysis}}$ Uncorrelated: $Z=[Z_1, Z_2, \dots, Z_p]$} \]

The PCR model is then

\[ y = Z\beta_z + \epsilon. \]

After centering and scaling \(X\), PCA is performed using singular value decomposition:

\[ X = UDV' \]

where \(V\) contains the principal component directions. The principal component matrix is

\[ Z = XV. \]

OLS on the original predictors is

\[ \hat{\beta}_x = (X'X)^{-1}X'y. \]

OLS on the principal components is

\[ \hat{\beta}_z = (Z'Z)^{-1}Z'y. \]

Using all principal components gives the same fitted values as OLS:

\[ \hat{y} = X\hat{\beta}_x = Z\hat{\beta}_z. \]

Therefore, PCR removes multicollinearity by using uncorrelated components, while keeping the same predictions when all components are used.

Comparison and Conclusion

The final comparison uses the same test data and the same two metrics for all predictive models: RMSE and test-set \(R^2\). Robust standard errors are not included as a separate predictive model because they change inference, not predictions.

  • The initial objective was to address possible multicollinearity among predictors.

  • After checking VIF/GVIF diagnostics, especially adjusted GVIF values, we found that multicollinearity was not a serious issue.

  • Thus, Ridge, LASSO, and PCR were used mainly to compare prediction accuracy with OLS.

  • Model diagnostics showed non-constant residual variance, so we shifted the focus toward heteroscedasticity.

  • We tried several model improvements, including:

    • subset model selection,
    • PCA subset selection,
    • different outlier filters,
    • quadratic terms,
    • cubic terms,
    • interaction terms,
    • combinations of main, interaction, quadratic, and cubic terms.

  • None of these approaches fully resolved the non-constant variance issue.

  • Therefore, the final model is more reliable for prediction than interpretation.

  • Coefficient interpretations and p-values should be treated with caution.

  • The final focus is prediction accuracy and model comparison, not formal inference.
